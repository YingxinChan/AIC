# SmartTrip AI Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the complete SmartTrip AI project skeleton — all folders, all screens (stubbed with placeholders), all API routes (stubbed with safe responses), Tailwind styling, and graceful error handling. No real business logic.

**Architecture:** Feature-based structure. `frontend/src/features/<domain>/` holds React pages + API callers per domain. `backend/routers/` (thin HTTP layer) calls `backend/services/` (business logic stubs). A standalone `ml/` directory holds offline training scripts. All routes and pages are stubbed so the full stack can be run and navigated immediately.

**Tech Stack:** React 18, Vite 5, Tailwind CSS 3, React Router v6, Axios, react-leaflet (frontend); FastAPI ≥ 0.111, SQLAlchemy 2 async, Pydantic v2, Celery, Redis, python-jose, passlib (backend); pytest, Vitest, @testing-library/react (tests).

## Global Constraints

- Python 3.11+; run all backend commands from within `backend/`
- React 18; Vite 5; Tailwind CSS 3; React Router v6; run all frontend commands from within `frontend/`
- All backend routes prefixed `/api`
- JWT stored as httpOnly cookie named `access_token`
- Skeleton auth: cookie absent → 401; cookie present → skip validation, return `{"id": 1, "email": "stub@example.com"}`
- Stub GET/DELETE → `{"status": "not_implemented"}`; POST/PUT/PATCH → `{"status": "not_implemented", "data": {}}`
- Placeholder UI: `<Placeholder label="..." />` — no fake/realistic data
- Error UI: `<ErrorMessage message="..." />` — no blank screens on API failure
- Destination hardcoded as London (51.5074, -0.1278) throughout — no city selection in MVP
- Map: Leaflet tile layer on Itinerary page, centred on London, no pins yet

> **Parallel execution note:** Tasks 1–7 (backend) and Tasks 8–12 (frontend) are independent and can be worked on simultaneously by different team members. Task 13 (ML directory) is independent of both.

```python
from fastapi import APIRouter, Response, Depends
from core.security import get_current_user
from schemas.auth import RegisterRequest, LoginRequest, AuthOut
from services import auth_service

router = APIRouter(prefix="/api/auth", tags=["auth"])

@router.post("/register", response_model=AuthOut)
async def register(body: RegisterRequest, response: Response):
    user = auth_service.register_user(body.email, body.password)
    response.set_cookie(key="access_token", value="stub-token", httponly=True)
    return {"user": user}

@router.post("/login", response_model=AuthOut)
async def login(body: LoginRequest, response: Response):
    user = auth_service.login_user(body.email, body.password)
    response.set_cookie(key="access_token", value="stub-token", httponly=True)
    return {"user": user}

@router.post("/logout", status_code=204)
async def logout(response: Response):
    response.delete_cookie("access_token")

@router.get("/me", response_model=AuthOut)
async def me(current_user: dict = Depends(get_current_user)):
    return {"user": current_user}
```

- [ ] **Step 4: Replace `frontend/src/features/auth/useAuth.js` with real implementation**

```jsx
import { createContext, useContext, useState, useEffect } from 'react'
import { getMe } from './authApi'

const AuthContext = createContext(null)

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    getMe()
      .then(data => setUser(data.user))
      .catch(() => setUser(null))
      .finally(() => setLoading(false))
  }, [])

  const login = (userData) => setUser(userData)
  const logout = () => setUser(null)

  return (
    <AuthContext.Provider value={{ user, loading, login, logout }}>
      {children}
    </AuthContext.Provider>
  )
}

export function useAuth() {
  const ctx = useContext(AuthContext)
  if (!ctx) throw new Error('useAuth must be used within AuthProvider')
  return ctx
}
```
